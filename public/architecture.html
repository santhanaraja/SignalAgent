Component Details

US
Universe Source

Assembles the 531-ticker candidate universe: 503 S&P 500 constituents (datahub CSV, Wikipedia fallback — the CSV's GICS column seeds the classifier for free) + 149 tickers from 21 ETF top-25 holdings (121 overlap), minus manual exclusions. Candidates cached 7 days; artifacts written atomically.

GC
GICS Classifier

Maps every candidate to a pure GICS sub-industry (126 currently). Resolution order: static seed map → 30-day on-disk cache → live yfinance lookup. A config alias map canonicalizes legacy/Yahoo labels at read time (cache keeps raw values, so alias edits are reversible and instant).

UB
Universe Builder

Friday 8PM ET rotation: ranks groups by median composite (0.5·YTD + 0.3·3M + 0.2·1M), selects top-15 groups × up to 7 quality tickers (score ≥50, mcap ≥$500M FX-converted, volume ≥500K/63d, history ≥90d; groups need ≥3 candidates). Week-key gates rebuilds; Monday self-heals a missed rotation. A viability floor (min groups/tickers/data coverage) aborts bad builds — the stale cache stays authoritative, with FALLBACK_INDUSTRY_GROUPS as the last-resort static net. Per-group near_misses embed the audit trail of who just missed and why.

SE
Signal Engine

Consumes the active dynamic universe (15 groups / 54 tickers this week). Computes RSI(14), MACD(12,26,9), MA20/50/200, momentum (1M/3M/YTD, 52-week range, trend strength N/20 = closes above same-day MA20), fundamentals, and per-group thesis-breaker status from macro proxies (clear → watch → warning → critical).

SC
Scoring & Trade Signals

score_stock(): base 50 plus five components — RSI (±15), MACD (±13), MA alignment (±14), YTD momentum (+12/−25 with overextension penalties), volume (±3) — clamped 0–100 with a score_components dict driving dashboard tooltips. Bands: ≥75 strong-buy · ≥60 buy · ≥45 hold · ≥30 sell · <30 strong-sell. compute_trade_signal(): phase-1 evidence tally, phase-2 first-match ladder where breaker critical → AVOID overrides everything. Weinstein stage is display-only. Full formulas: docs/scoring.md.

RG
Regime Gauge — Trend Chassis (Gauge B, D-008)

The 200DMA trend state DECIDES direction; VIX 5-day, HY OAS 60-day percentile, and RSP/SPY breadth are throttle modifiers that scale exposure within the trend, never override it. Locked calibration (evidence-chosen, 2,896-day replay pin): throttle fires at VIX ≥ 22, HY pctile ≥ 90, breadth < −0.5 (any one); asymmetric hysteresis — downgrades instant, upgrades need 2 consecutive closes; state recomputed by causal replay of the trailing 60 sessions each run. Chassis state → regime label → R28's 90/50/25/5 ceiling. The Build-1A voter parliament remains behind the config engine flag (revert lever). References: gauge-b-campaign.md · D-008.

PS
Position Signal Engine (Build 1B)

Per-ticker exit/re-entry state machine over positions.json (authoritative holdings + watchlist): HELD → EXIT_FIRED → WATCHING → RE_ENTRY_ARMING → RE_ENTRY_READY. Re-entry needs all five conditions — SMA20 reclaim, 2-close or 0.5×ATR(14) confirmation, regime gate (Choppy = "A+ only"), flat-or-rising SMA20, GICS group in the current universe (unmapped names pass but are flagged; outages fail closed). Every watcher carries the D-011 A+ grade — seven computed rows (conditions, extension, approach filter, RSI 45–70, score ≥75, breaker clear, ≥15-session earnings runway strictly before the print); a knife grades C in every regime, and Choppy hard-gates READY without A+ (rendered blocked). Reports extension_pct / extension_atr per ticker. Strictly close-basis; live stop shown for HELD. Signals only — nothing auto-executes.

The rule layer finally reads real money: positions.json + live closes against account_capital_usd. HM
History & Events

history_manager diffs consecutive runs into 8 event types (signal_change, score_change, group_signal_change, group_rank_change, group/ticker add/remove) with severity levels, plus timestamped snapshots. The 9th type — position_state_change — is written by the position engine to a separate position_events.json (single-writer constraint: history_manager rewrites history.json wholesale, so co-writing would race). history.html merges both files into one filterable timeline.

DF
Defense Layers

Every JSON write site (signal engine, universe builder, framework runner + regime history, position events) passes through sanitize_for_json — non-finite floats become null and each replacement is logged with its key path, so a NaN can never blank a browser page silently. The regime gate fails closed on missing data; the position engine skips (never wipes) state on unreadable input; the universe builder's viability floor keeps the last good universe over a bad build; membership churn is always evented, never silent.

CD
Automation & Deploy

Compute happens in GitHub Actions: update-signals (cron */15 during market hours — throttled by GitHub's scheduler to roughly hourly in practice) runs signal engine → history manager → framework and commits the JSON artifacts; rotate-universe fires Saturday 01:00 UTC (= Friday 8PM ET) plus manual workflow_dispatch, serialized against the signals job. Every push auto-deploys the Render web service, where Flask (ticker_api.py) serves all /api/* endpoints and the static pages. The repo is the data bus.

Data Flow

Weekly (Fri 8PM ET): S&P 500 + ETF holdings → universe_source → 531 candidates → gics_classifier (126 sub-industries) → universe_builder → top-15 groups × ≤7 tickers → universe_active.json (serves the following week; Monday self-heal)
~Hourly (market hours): active universe → signal_engine → indicators + momentum + fundamentals + breakers → score_stock() (5 components) → signal bands → compute_trade_signal() (tally + ladder) → signals.json → history_manager → 8 event types + snapshots
Framework (same run): regime gauge (trend chassis + throttles + macro_inputs) → legacy theme ranker (display-only, retiring) → position engine (5 conditions + A+ grade → position_events.json) → rule engine R1–R27 (active state = holdings' groups; R5→R28) → R28 dollar caps → framework.json (incl. universe_leadership)
Saturday rotation: the universe gate re-reads (top-15 selected groups; weeks_in_universe advances) — legacy theme Sunday review continues display-only until Phase 3
Serving: CI commits JSON → Render auto-deploys → Flask serves /api/* + static pages; history.html merges history.json + position_events.json into one timeline
